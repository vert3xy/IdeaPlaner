# 🔌 Спецификация API (RESTful)

> [!NOTE]  
> Полная интерактивная документация (Swagger UI / OpenAPI) генерируется автоматически и доступна по адресу `http://localhost:8000/docs` после запуска приложения.
> В данном документе описаны ключевые эндпоинты, реализующие сложную бизнес-логику (EAV-паттерн и динамические фильтры).

## 🛡️ Аутентификация
Все описанные ниже методы требуют авторизации. В заголовке запроса необходимо передавать JWT-токен.
**Header:** `Authorization: Bearer <your_access_token>`

---

## 1. Умные фильтры категорий

🟢 **`GET`** `/categories/{cat_id}/filters`

**Описание:** Возвращает конфигурацию фильтров для указанной категории. Если передана конкретная категория, система "на лету" анализирует таблицу `ideas` и извлекает уникальные значения из JSON-колонки `attributes`, формируя массив `dynamic_filters`.

### 📌 Параметры пути (Path Parameters)
| Параметр | Тип | Описание | Обязательный |
| :--- | :--- | :--- | :---: |
| `cat_id` | `string` | ID категории (число) или строка `"all"` (для получения только общих фильтров) | Да |

### 📤 Успешный ответ (HTTP 200 OK)

> [!IMPORTANT]  
> **Бизнес-правило:** Массив `common_filters` возвращается всегда (Статус, Дата, Автор). Массив `dynamic_filters` генерируется только если передан валидный числовой ID категории.

<details>
<summary><b>Пример ответа: генерация фильтров для категории "IT" (Нажми, чтобы развернуть)</b></summary>

```json
{
  "category_id": 1,
  "category_label": "IT Проекты",
  "dynamic_filters": [
    {
      "name": "programming_language",
      "label": "Язык программирования",
      "type": "select",
      "values": [
        "Python",
        "JavaScript",
        "Go"
      ]
    }
  ],
  "common_filters": [
    {
      "name": "status",
      "label": "Статус",
      "type": "select",
      "values": ["new", "planned", "done", "rejected"]
    },
    {
      "name": "created_at",
      "label": "Дата добавления",
      "type": "date"
    }
  ]
}
```
</details>

### ❌ Возможные ошибки
* `HTTP 400 Bad Request`: Некорректный формат ID (например, передана строка вместо числа, отличная от "all").
* `HTTP 404 Not Found`: Категория с указанным ID не найдена в базе данных.

## 2. Создание идеи с динамическими полями

🔵 **`POST`** `/ideas/`

**Описание:** Создает новую идею. Ожидает базовые поля и словарь `attributes` произвольной структуры, зависящий от выбранной категории.

### 📥 Тело запроса (Request Body - `application/json`)
| Поле | Тип | Описание | Обязательный |
| :--- | :--- | :--- | :---: |
| `title` | `string` | Краткое название идеи | Да |
| `description` | `string` | Подробное текстовое описание | Нет |
| `category_id` | `integer` | ID категории, к которой относится идея | Да |
| `attributes` | `object` | Словарь ключ-значение с динамическими полями | Да (может быть пустым `{}`) |

<details>
<summary><b>Пример тела запроса (Payload)</b></summary>

```json
{
  "title": "Интеграция с Telegram",
  "description": "Сделать бота для уведомлений",
  "category_id": 2,
  "attributes": {
    "budget": "5000",
    "deadline": "2024-11-01",
    "is_priority": "true"
  }
}
```
</details>

### 📤 Успешный ответ (HTTP 200 OK)
Возвращает Pydantic-схему `IdeaOut`, содержащую сгенерированный `id`, статус (по умолчанию `new`), дату создания и данные об авторе из JWT-токена.

---

## 3. Изменение статуса идеи (Жизненный цикл)

🟡 **`PATCH`** `/ideas/{idea_id}/status`

**Описание:** Легковесный метод для перевода идеи на следующий этап (State Machine). Обновляет только поле статуса и автоматически триггерит обновление `updated_at` в базе данных.

### 📌 Параметры пути и тело запроса
* **Path:** `idea_id` (integer) — уникальный идентификатор идеи.
* **Body:** JSON-объект со статусом.

```json
{
  "status": "planned" 
}
```
*(Допустимые значения: `new`, `planned`, `done`, `rejected`)*

### ❌ Возможные ошибки
* `HTTP 404 Not Found`: Идея с переданным `idea_id` не существует.
* `HTTP 422 Unprocessable Entity`: Передано невалидное тело запроса (отсутствует поле status).
