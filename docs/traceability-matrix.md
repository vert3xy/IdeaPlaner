# 🏗 Нефункциональные требования (NFR)

---

## 🔐 1. Безопасность (Security)

| Требование | Описание | Реализация |
| :--- | :--- | :--- |
| **Хранение паролей** | Пароли пользователей никогда не должны храниться в открытом виде. | Использование библиотеки `bcrypt` для хеширования с солью (файл `auth.py`). |
| **Аутентификация** | Система должна поддерживать архитектуру Stateless для защиты сессий. | Реализована на базе **JWT (JSON Web Tokens)** с использованием алгоритма HS256. |
| **Защита данных** | Секретные ключи приложения не должны находиться в коде. | Вынос `SECRET_KEY` и параметров БД в переменные окружения (`.env`). |
| **Авторизация** | Операции удаления и редактирования доступны только владельцу контента. | Проверка соответствия `author_id` в токене и записи БД в роутерах. |

---

## ⚡ 2. Производительность (Performance)

*   **Скорость отклика:** Время обработки API-запроса на получение списка идей не должно превышать 200мс (при объеме данных до 10 000 записей).
*   **Оптимизация БД:** 
    *   Использование индексов в SQLite для полей `author_id`, `category_id` и `status` (файл `models.py`).
    *   Применение `joinedload` в SQLAlchemy для исключения проблемы N+1 при загрузке идей вместе с авторами.
*   **Frontend-оптимизация:** Минимальное использование внешних библиотек (Vanilla JS) обеспечивает мгновенную загрузку UI без тяжелых фреймворков.

---

## 📈 3. Масштабируемость и Портируемость (Scalability & Portability)

> [!IMPORTANT]
> Система спроектирована с учетом возможного роста и смены технологического стека.

1.  **DB Agnostic:** Благодаря использованию **SQLAlchemy ORM**, текущая база данных SQLite может быть заменена на PostgreSQL или MySQL изменением одной строки подключения без переписывания бизнес-логики.
2.  **Контейнеризация:** Приложение упаковано в **Docker**. Это гарантирует, что система будет работать идентично на локальном компьютере, сервере разработки или в облаке.
3.  **Гибкая метамодель:** Хранение атрибутов в формате JSON позволяет добавлять новые типы данных без проведения миграций структуры БД.

# ⚖️ Бизнес-правила (Business Rules)

## 🔑 1. Безопасность и Доступ (Access Control)

| ID | Правило | Описание |
| :--- | :--- | :--- |
| **BR-SEC-01** | **Обязательная авторизация** | Доступ к созданию, просмотру и изменению идей возможен только при наличии валидного JWT-токена. |
| **BR-SEC-02** | **Владение данными** | Пользователь имеет право удалять или изменять статус только тех идей, автором которых он является (`author_id` в БД должен совпадать с `sub` в токене). |
| **BR-SEC-03** | **Уникальность логина** | Регистрация двух пользователей с одинаковым `username` запрещена. |

---

## 📋 2. Жизненный цикл идеи (Lifecycle)

| ID | Правило | Описание |
| :--- | :--- | :--- |
| **BR-LIFE-01** | **Начальный статус** | При создании новой идеи ей автоматически присваивается статус `new`. |
| **BR-LIFE-02** | **Непрерывность истории** | При любом изменении статуса или описания поле `updated_at` должно обновляться текущим временем (UTC). |
| **BR-LIFE-03** | **Модель переходов** | Система поддерживает 4 состояния: `new`, `planned`, `done`, `rejected`. Переходы между ними свободные, но логически завершенная идея (`done` или `rejected`) не должна возвращаться в `new`. |

---

## 🛠 3. Логика атрибутов (Data Logic)

*   **BR-DATA-01 (Связь Категория-Атрибут):** Каждая категория может иметь неограниченное количество привязанных атрибутов, но один и тот же атрибут может использоваться в разных категориях (связь Many-to-Many).
*   **BR-DATA-02 (Типизация JSON):** Значения атрибутов в поле `attributes` (JSON) должны соответствовать типам, указанным в таблице `attributes` (string, select, date), однако на уровне БД SQLite валидация типов не производится (реализуется на уровне приложения).
*   **BR-DATA-03 (Уникальность категорий):** Системное имя категории (`name`) должно быть уникальным и написанным в стиле `snake_case` для корректной работы API.

## 🧩 Структура поля `attributes` (JSON)
Поле `attributes` в таблице `ideas` не имеет жесткой схемы. Его структура определяется набором атрибутов, связанных с выбранной `category_id`.

**Пример для категории "IT-Проект" (ID=1):**
```json
{
  "tech_stack": "Python, FastAPI",
  "estimated_hours": 40,
  "is_open_source": true
}
