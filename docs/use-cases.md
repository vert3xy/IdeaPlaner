## 📝 UC-01: Создание идеи с динамическими атрибутами

> [!NOTE]  
> **Бизнес-ценность:** Позволяет пользователям описывать идеи разных категорий (например, "IT-проект" или "Крупная покупка") с абсолютно индивидуальным набором полей. При этом структура таблиц в базе данных остается неизменной (используется JSON-колонка `attributes`).

| Характеристика | Описание |
| :--- | :--- |
| **Актор** | 👤 Авторизованный пользователь |
| **Триггер** | Нажатие кнопки «Добавить идею» на главной странице |
| **Предусловия** | Пользователь успешно авторизован, в системе заведена хотя бы одна категория с привязанными атрибутами (`linked_attributes`). |

🟢 Основной сценарий (Happy Path)
Пользователь инициирует создание идеи.
Фронтенд запрашивает у API доступные категории и связанные с ними поля ввода.
Пользователь выбирает категорию из выпадающего списка.
Интерфейс мгновенно перерисовывается, добавляя специфичные поля для выбранной категории.
Пользователь заполняет обязательное поле title (Название), description (Описание) и сгенерированные динамические поля.
При нажатии на кнопку "Сохранить", система упаковывает динамические поля в единый JSON-объект.
Система отправляет запрос к API на создание записи.
Бэкенд сохраняет идею в БД со статусом по умолчанию new и привязывает её к author_id текущего пользователя.
Идея появляется в верхней части дашборда пользователя.
🔴 Альтернативные сценарии и обработка ошибок
[!WARNING]
Обработка исключительных ситуаций системой:
[A1] Категория удалена или не существует: Если в момент отправки формы переданный category_id не найден в БД (например, был удален администратором в другой сессии), API прерывает транзакцию и возвращает HTTP 400: Категория не найдена. Пользователь видит всплывающее уведомление об ошибке.
[A2] Ошибка валидации данных: Если пользователь не заполнил обязательное поле title (название), Pydantic на стороне бэкенда отклонит запрос со статусом HTTP 422 Unprocessable Entity. Форма не закроется, поля с ошибками будут подсвечены красным.
[A3] Отмена действия: До момента нажатия кнопки "Сохранить" пользователь может закрыть модальное окно. Данные очищаются, запрос к API не выполняется.
